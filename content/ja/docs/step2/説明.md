# Step2: 実践的開発フロー演習の設計

GitHub Copilotを活用した実際のプロジェクト開発における一貫した流れを実践する演習設計です。

## 全体のコンセプト

**対象プロジェクト**: **Todo管理アプリケーション**（React + TypeScript + Node.js）
- フロントエンド: React + TypeScript + Vite
- バックエンド: Node.js + Express + TypeScript
- データベース: SQLite（簡単なセットアップのため）
- テスト: Jest + React Testing Library

---

## Example-1: プロジェクトの構造及びコード理解

### 内容
- **既存プロジェクトの解析**: Copilotを使ったコードベース理解
- **アーキテクチャ把握**: システム全体の構造理解
- **依存関係の分析**: package.json、設定ファイルの理解

### 実践内容
1. **プロジェクト概要の把握**
   - `#workspace` を使った全体構造の理解
   - Copilot Chat での「このプロジェクトの概要を教えて」
   - 主要なファイル・フォルダの役割説明

2. **コード流れの理解**
   - 特定機能のフロー追跡（例: Todo作成処理）
   - データの流れ（API → Service → Component）
   - 状態管理の仕組み理解

3. **技術スタックの分析**
   - 使用しているライブラリ・フレームワークの確認
   - 設定ファイルの内容理解
   - 開発環境の構築方法理解

### 学習目標
- 既存コードベースを効率的に理解する手法
- Copilotを使った技術的負債の発見
- プロジェクト参画時の立ち上がり高速化

## 1. ツールセットとは？

ツールセットは、チャットで使用できる **関連するツールのコレクション** です。複数のツールをひとまとめにして、特定の作業に最適化されたツールグループを作成できます。

> **ポイント**
>
> * **効率的なツール管理** … 関連するツールをグループ化して整理
> * **チャットから直接参照** … 例：`#myToolset このプロジェクトを分析して`
> * **エージェントモードで選択可能** … ツールピッカーから任意のツールセットを選択
> * **カスタムワークフロー** … プロジェクトや作業内容に応じたツール構成

---

## 2. ツールセットファイルの構造

ツールセットは `.toolsets.jsonc` ファイルで定義します：

```jsonc
{
  "ツールセット名": {
    "tools": [
      "ツール1",
      "ツール2",
      "ツール3"
    ],
    "description": "ツールセットの説明",
    "icon": "アイコン名"
  }
}
```

### 2.1 各プロパティの説明

| プロパティ | 説明 | 例 |
|------------|------|-----|
| `tools` | 含めるツール名のリスト | `["changes", "codebase", "problems"]` |
| `description` | ツールセットの簡単な説明 | `"コードレビュー用ツールセット"` |
| `icon` | ツールピッカーで表示するアイコン | `"search"`, `"debug"`, `"tools"` など ([アイコン集](https://code.visualstudio.com/api/references/icons-in-labels)) |

---

## 3. ツールセットの活用シーン

| シーン | ツールセット例 | 含まれるツール |
|--------|----------------|----------------|
| **コードレビュー** | `reviewer` | `#changes`, `#problems`, `#codebase`, `#usages` |
| **デバッグ作業** | `debugger` | `#problems`, `#runTests`, `#terminalSelection`, `#testFailure` |
| **新規開発** | `developer` | `#new`, `#editFiles`, `#runTasks`, `#openSimpleBrowser` |
| **外部調査** | `researcher` | `#fetch`, `#githubRepo`, `#extensions`, `#VSCodeAPI` |

## 4. ツールセットの作成方法

### 4.1 設定画面からの作成
![ツールの選択方法](../images/toolset_create.png)
1. **チャットビューを開く** → 右上の **「歯車」** ボタンをクリック
2. **「ツールセット」** を選択
3. **「新しいツールセットファイル」** を選択
4. **ファイル名を入力**（例：`my-toolset.jsonc`）
5. `.toolsets.jsonc` ファイルが開かれるので、内容を編集

### 4.2 コマンドパレットからの作成
![ツールの選択方法](../images/toolset_create2.png)
1. `⇧⌘P` (Mac) / `Ctrl+Shift+P` (Windows) でコマンドパレットを開く
2. **「チャット：ツールセットの構成」** を実行
3. ツールセットファイルが開かれる

## 5. 実践例：カスタムツールセットを作成

### :pen: 例題

外部調査に特化したツールセット `researcher` を作成してみましょう。

```jsonc
{
  "researcher": {
    "tools": [
      "fetch",
      "githubRepo",
      "extensions",
      "vscodeAPI",
      "codebase",
      "search"
    ],
    "description": "外部情報の調査・収集用のツールセット",
    "icon": "search"
  }
}
```

### :robot: 使用例

ツールセット作成後、チャットで以下のように使用できます：

```text
#researcher Next.jsの最新のスタイリング手法について調べて、当プロジェクトに適用できそうな方法を提案してください
```

Copilot は `researcher` ツールセットに含まれるツールを使って：
1. `#fetch` で公式ドキュメントやブログの最新情報を取得
2. `#githubRepo` で人気プロジェクトの実装例を検索
3. `#codebase` で現在のプロジェクト構成を分析
4. `#extensions` で関連するVS Code拡張機能を調査

## 6. ツールセットの管理Tips

### 6.1 目的別ツールセットの例

**フロントエンド開発用**
```jsonc
{
  "frontend": {
    "tools": ["editFiles", "openSimpleBrowser", "runTasks", "problems"],
    "description": "フロントエンド開発とプレビュー",
    "icon": "browser"
  }
}
```

**API開発用**
```jsonc
{
  "api": {
    "tools": ["editFiles", "runTests", "fetch", "terminalSelection"],
    "description": "API開発とテスト",
    "icon": "server"
  }
}
```

### 6.2 ツールセットの切り替え

- **エージェントモード**：ツールピッカー（🔧）からツールセットを選択
- **通常チャット**：`#ツールセット名` で直接指定
- **プロンプトファイル**：事前定義されたプロンプトでツールセットを活用

---

## 7. ベストプラクティス

### 7.1 命名規則
- **目的が分かりやすい名前** を使用（例：`reviewer`, `debugger`, `researcher`）
- **短く覚えやすい** 名前にする

### 7.2 ツール選択
- **関連性の高いツール** をグループ化
- **よく一緒に使うツール** をまとめる
- **5-8個程度** に絞って使いやすさを重視

### 7.3 説明文
- **具体的な用途** を記載
- **どんな作業に適しているか** を明示

---

## :memo: 練習

1. **自分の作業スタイルに合わせたツールセット** を作成してみてください
2. **既存の組み込みツールを組み合わせて** 、特定のタスク用のツールセットを設計してください
3. **プロジェクトの種類別** （React、Python、Node.js など）にツールセットを作成してみてください

---

## 8. まとめ

* ツールセットは **関連ツールをグループ化** し、効率的な作業環境を構築する機能
* **`.toolsets.jsonc`** ファイルで簡単に定義・カスタマイズ可能
* **チャットから直接呼び出し** または **エージェントモードで選択** して使用
* **作業の性質に応じたツールセット** を作成することで、開発効率が大幅に向上

GitHub Copilot Chat では、**チャット参加者 (Chat Participants)**、**スラッシュコマンド (Slash Commands)**、**チャット変数 (Context Variables)** の 3 つを組み合わせることで、**短い記述**だけでコード修正・ドキュメント追加・テスト生成などを効率的に指示できます。ここでは、それぞれの概要や活用方法を示し、最後に複数のショートカットを組み合わせて使う方法も紹介します。

---

## 1. チャット参加者 (Chat Participants)

チャットの先頭に **`@xxx`** を入力することで、Copilot に対して「この領域の情報を優先的に参照してほしい」と指示できます。

| コマンド        | 説明                                                                        |
|-----------------|-------------------------------------------------------------------------------|
| **@workspace**  | ワークスペース全体のファイル・依存関係を文脈として参照し、より精度の高い提案を得る |
| **@terminal**   | ターミナルでの操作やコマンド出力を文脈として扱い、シェルコマンドやログ解析が便利   |
| **@vscode**     | Visual Studio Codeの設定やコマンドにフォーカスしてヘルプや提案を得る           |
| **@github**     | GitHubリポジトリやIssue/PRに関する文脈を優先し、操作やAPIに関する提案が可能      |

---

## 2. スラッシュコマンド (Slash Commands)

チャット欄で **`/`** を入力すると表示される、**よく使う操作**を簡単に呼び出すためのコマンド群です。  
下記のような操作を組み合わせることで、**修正 / ドキュメント / テスト生成**などをシンプルに指示できます。

| コマンド        | 説明                                                                                  |
|-----------------|-----------------------------------------------------------------------------------------|
| **/fix**        | 選択したコードやファイル全体をリファクタ・修正                                            |
| **/doc**        | ドキュメンテーションコメント (JavaDoc など) を自動生成                                    |
| **/tests**      | テストコードを自動生成する                                                               |
| **/explain**    | コードの動作や目的を説明                                                                 |
| **/new**        | 新規ファイル/コードを作成                                                                |
| **/newChat**    | チャット履歴をリセットして、新規スレッドの会話に切り替える                                 |
| **/help**       | スラッシュコマンドの一覧・使い方を表示                                                   |
| **/api**        | VS Codeの拡張開発用のAPIに関する質問を行う                                               |
| **/newNotebook**| Jupyter Notebookを新規作成 (Notebook利用者向け)                                         |

---

## 3. チャット変数 (Context Variables)

「**#**」を先頭につけて**ファイル指定**や**選択範囲**などの文脈を Copilot に渡せる機能です。  
これにより、**どのファイル / どの範囲**を優先して分析・修正・説明するかを**明示**できます。

| コマンド               | 説明                                                                           |
|-----------------------|-------------------------------------------------------------------------------|
| **#selection**        | エディタで選択したコードブロックだけを文脈として扱う                            |
| **#terminalSelection**| ターミナルで選んだログや出力だけを文脈として扱う                                 |
| **#file:XXX**         | 指定したファイルを読み込み、修正や解説などを行う                                 |
| **#codebase**         | プロジェクト全体を文脈として参照                                                 |
| **#terminalLastCommand** | ターミナルで最後に実行したコマンドを文脈に含める                             |

---

## 4. ショートカットコマンドの組み合わせ例

### 4-1. 1行で組み合わせる

```plaintext
@workspace /fix #file:xxx.java
```
- **@workspace** … プロジェクト全体の構造を踏まえた提案  
- **/fix** … 修正・リファクタの提案を即時に実行  
- **#file:xxx.java** … あるJavaファイルに対する背景情報を提示  

### 4-2. 選択範囲でピンポイント修正

1. **エディタでメソッド本体を選択**  
2. ショートカットでインラインチャット呼び出し (Ctrl+Shift+I etc.)  
3. 入力:  
   ```plaintext
   #selection /fix  
   このループをストリームAPIで書き直して
   ```
   - これで選択範囲だけに対する修正案を得られる

### 4-3. テスト生成を同時に依頼

```plaintext
@workspace /tests #file:xxx.java
```
- あるJavaのコードに対する**ユニットテスト**を**自動生成**  
- 大規模なファイルなら範囲選択→`#selection` でも可

---

## まとめ

- **チャット参加者 (@xxx)**: Copilot に「ワークスペース」「ターミナル」「VSCode設定」など特定領域を優先させる  
- **スラッシュコマンド (/xxx)**: コード修正やドキュメント生成など、**よく使う操作**を手短に呼び出し  
- **チャット変数 (#xxx)**: ファイルや選択範囲、ターミナル出力などを**文脈**として与え、**正確な提案**を引き出す  
- **組み合わせ**で 1行コマンドを作り、「どの領域をどう修正/解説/生成するか」を簡潔に表現 → **作業時間**が大幅に短縮

こうした**Copilot Chat**のショートカットコマンドを活用すると、**エディタ内**での処理が非常に高速・スムーズになり、**開発効率**や**開発者体験**が大きく向上します。

これまでの基本テクニックを組み合わせて使って、業務で活用できるシナリオをいくつか試してみましょう。

膨大な情報の中から必要な情報を選び正しく理解することは重要です。設計書だけでなく、専門知識や、サプライヤーとのメールでのやり取りなども多岐にわたる情報を効率よく理解する必要があります。

要約を行うときは、正確な情報を与え、観点を追加することでより意図した近い結果が得られます。その他の具体的な指示内容として、回答の文字数指定、単語や文、段落や箇条書きなどの回答形式の設定などがあります。

> この文章は「[Azure OpenAI On Your Data](https://learn.microsoft.com/ja-JP/azure/ai-services/openai/concepts/use-your-data?tabs=ai-search)」から引用しています。

次のプロンプトを入力します。

```text
3重の$で区切られた文章を、100文字程度 で要約してください。

$$$
Azure OpenAI On Your Data では、モデルのトレーニングや微調整をすることなく、GPT-35-Turbo や GPT-4 などの高度な AI モデルを独自のエンタープライズ データ上で実行できます。 より正確にデータをチャットして分析することができます。 指定したデータ ソースで使用可能な最新情報に基づいて、回答をサポートするソースを指定できます。 Azure OpenAI On Your Data には、REST API、SDK、または Azure OpenAI Studio の Web ベースのインターフェイスを使用してアクセスできます。 また、データに接続して拡張チャット ソリューションを有効にする Web アプリを作成したり、Copilot Studio (プレビュー) で Copilot として直接デプロイしたりすることもできます。
$$$
```

観点を入れて要約してみましょう。
次のプロンプトを入力します。

読み手を想定して文章に要約してみましょう。「小学生」以外にも、「IT業界に入社したばかりの新入社員」や「ベテランのプログラマー」「英語が母国語のUSのサプライヤー」などでどのように変わるか確認してください。
